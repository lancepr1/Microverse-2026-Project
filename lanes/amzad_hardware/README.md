# Amzad: hardware-box exploration + DT visual realism

## What it does
Two threads. (1) A small multi-sensor prototype (motion, camera, mic, humidity
to a Pi or phone) exploring richer environmental fingerprints. Exploratory, not
path-critical. (2) Server-rack and IPDVS-style visual elements added to
Hendricks's Blender scene.

## How to run
For visual contributions, add geometry to Hendricks's scene and keep object
naming consistent so state variables still resolve. Coordinate directly with
Hendricks ("I built a thing, does it fit your scene").

## Norm (important)
Do not bring any data, footage, or images from prior NYC DOE camera deployments
into this project. Those people did not consent to a federal research project.
Synthetic or stock imagery only.

## Who to ask
Dr. Qu (hardware architecture), Lance (code and integration), Hendricks (scene
fit).

## Week-1 deliverable
(1) hardware-box architecture decision (Pi vs phone) with reasoning, parts list,
target sensors, prototype-by-week-3 schedule; (2) one-page visual reference doc
for Hendricks using synthetic or stock imagery.
